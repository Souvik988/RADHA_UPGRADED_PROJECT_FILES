# Phase BE-17: Bulk Scan Processing

## Phase Metadata

- **Phase ID**: BE-17
- **Phase Name**: Bulk Scan Processing
- **Section**: Backend Execution — Audit & Compliance Layer
- **Depends On**: BE-01 to BE-16
- **Blocks**: BE-18, mobile offline sync
- **Estimated Duration**: 2 days

## Goal

Implement production-grade bulk scan processing for offline-sync scenarios: idempotency via client-generated UUIDs, batch validation, conflict resolution, async processing for large batches (1000+ scans), progress tracking, partial success handling, and audit trail for sync events.

## Why This Phase Matters

Mobile users scan offline, then sync:
- Auditor in basement (no signal) scans 200 items
- Returns to internet → 200 items must sync
- Must handle: duplicates, conflicts, partial failures
- Must be idempotent (retries don't double-record)
- Must complete within reasonable time

Without proper bulk processing:
- Mobile users lose data on sync failures
- Duplicate scans created from retries
- Server timeouts on large batches
- Poor offline UX

### 2. Idempotency Service (Redis-backed)

```typescript
// server/src/modules/scans/services/idempotency.service.ts
import { Injectable, OnModuleInit } from '@nestjs/common';
import Redis from 'ioredis';
import { ConfigService } from '../../../config/config.service';
import { ScanItemsRepository } from '../repositories/scan-items.repository';
import { IIdempotencyService } from '../types/sync.types';

@Injectable()
export class IdempotencyService implements IIdempotencyService, OnModuleInit {
  private redis!: Redis;
  private readonly KEY_PREFIX = 'scan:idem:';
  private readonly TTL_SECONDS = 7 * 24 * 60 * 60; // 7 days

  constructor(
    private readonly config: ConfigService,
    private readonly itemsRepo: ScanItemsRepository,
  ) {}

  onModuleInit() {
    this.redis = new Redis({
      host: this.config.redis.host,
      port: this.config.redis.port,
      password: this.config.redis.password,
      db: this.config.redis.db,
      keyPrefix: this.config.redis.keyPrefix,
    });
  }

  async isProcessed(clientId: string, sessionId: string): Promise<boolean> {
    const key = this.buildKey(clientId, sessionId);
    const exists = await this.redis.exists(key);
    return exists === 1;
  }

  async markProcessed(
    clientId: string,
    sessionId: string,
    scanItemId: string,
  ): Promise<void> {
    const key = this.buildKey(clientId, sessionId);
    await this.redis.setex(key, this.TTL_SECONDS, scanItemId);
  }

  async findExisting(clientId: string, sessionId: string): Promise<ScanItem | null> {
    const key = this.buildKey(clientId, sessionId);
    const scanItemId = await this.redis.get(key);
    
    if (!scanItemId) {
      // Fallback: check DB by metadata
      return this.itemsRepo.findByClientId(clientId, sessionId);
    }
    
    return this.itemsRepo.findById(scanItemId);
  }

  private buildKey(clientId: string, sessionId: string): string {
    return `${this.KEY_PREFIX}${sessionId}:${clientId}`;
  }
}
```

### 3. Bulk Scan Service

```typescript
// server/src/modules/scans/services/bulk-scan.service.ts
import { Injectable, Logger } from '@nestjs/common';
import { InjectQueue } from '@nestjs/bull';
import { Queue } from 'bull';
import { v4 as uuidv4 } from 'uuid';
import { DbService } from '../../../db/db.service';
import { ScanSyncBatchesRepository } from '../repositories/scan-sync-batches.repository';
import { ScanSessionsRepository } from '../repositories/scan-sessions.repository';
import { IdempotencyService } from './idempotency.service';
import { ScanItemService } from './scan-item.service';
import { LoggerService } from '../../../logging/logger.service';
import {
  IBulkScanService,
  BulkSyncDto,
  BulkSubmissionResult,
  BulkSyncStatus,
  BulkProcessResult,
  BulkScanItemDto,
  BulkSyncError,
} from '../types/sync.types';
import {
  NotFoundException,
  BusinessException,
} from '../../../common/errors/business.exception';
import { ErrorCode } from '../../../common/errors/error-codes';

@Injectable()
export class BulkScanService implements IBulkScanService {
  private readonly logger = new Logger(BulkScanService.name);
  
  // Threshold: smaller batches process synchronously
  private readonly SYNC_THRESHOLD = 50;
  private readonly MAX_BATCH_SIZE = 5000;

  constructor(
    private readonly db: DbService,
    private readonly batchesRepo: ScanSyncBatchesRepository,
    private readonly sessionsRepo: ScanSessionsRepository,
    private readonly idempotency: IdempotencyService,
    private readonly itemService: ScanItemService,
    private readonly logger2: LoggerService,
    @InjectQueue('bulk-scan') private readonly queue: Queue,
  ) {}

  async submitBatch(
    dto: BulkSyncDto,
    userId: string,
  ): Promise<BulkSubmissionResult> {
    // Validate
    if (dto.items.length === 0) {
      throw new BusinessException(ErrorCode.INVALID_INPUT, 'No items to sync');
    }
    if (dto.items.length > this.MAX_BATCH_SIZE) {
      throw new BusinessException(
        ErrorCode.INVALID_INPUT,
        `Batch too large. Max ${this.MAX_BATCH_SIZE} items.`,
      );
    }

    // Verify session ownership
    const session = await this.sessionsRepo.findById(dto.sessionId);
    if (!session) throw new NotFoundException('ScanSession', dto.sessionId);
    if (session.userId !== userId) {
      throw new BusinessException(ErrorCode.FORBIDDEN, 'Not your session');
    }
    if (session.status !== 'active') {
      throw new BusinessException(
        ErrorCode.SCAN_SESSION_CLOSED,
        'Session must be active for bulk sync',
      );
    }

    // Decide: sync or async?
    if (dto.items.length <= this.SYNC_THRESHOLD) {
      // Process synchronously
      const result = await this.processSync(dto.items, userId);
      return {
        batchId: 'sync',
        status: 'completed',
        totalItems: dto.items.length,
        estimatedDurationSeconds: 0,
      };
    }

    // Process asynchronously via queue
    const batch = await this.batchesRepo.create({
      sessionId: dto.sessionId,
      userId,
      status: 'queued',
      totalItems: dto.items.length,
      deviceId: dto.metadata?.deviceId,
      appVersion: dto.metadata?.appVersion,
      metadata: dto.metadata,
    });

    // Add to Bull queue
    await this.queue.add('process-batch', {
      batchId: batch.id,
      sessionId: dto.sessionId,
      userId,
      items: dto.items,
    }, {
      attempts: 3,
      backoff: {
        type: 'exponential',
        delay: 5000,
      },
      removeOnComplete: 100,
      removeOnFail: 500,
    });

    return {
      batchId: batch.id,
      status: 'queued',
      totalItems: dto.items.length,
      estimatedDurationSeconds: Math.ceil(dto.items.length * 0.3),
    };
  }

  async processSync(
    items: BulkScanItemDto[],
    userId: string,
  ): Promise<BulkProcessResult> {
    const startTime = Date.now();
    const successful: any[] = [];
    const duplicates: BulkScanItemDto[] = [];
    const failed: BulkSyncError[] = [];

    for (const item of items) {
      try {
        // Check idempotency
        const sessionId = (item as any).sessionId; // From parent context
        const existing = await this.idempotency.findExisting(item.clientId, sessionId);
        if (existing) {
          duplicates.push(item);
          continue;
        }

        // Process scan
        const result = await this.itemService.recordScan(
          sessionId,
          {
            ean: item.ean,
            scannedAt: item.scannedAt,
            expiryDate: item.expiryDate,
            manufactureDate: item.manufactureDate,
            batchNumber: item.batchNumber,
            quantity: item.quantity,
            shelfLocation: item.shelfLocation,
            notes: item.notes,
            imageMediaId: item.imageMediaId,
            latitude: item.latitude,
            longitude: item.longitude,
          },
          userId,
        );

        // Mark idempotent
        await this.idempotency.markProcessed(
          item.clientId,
          sessionId,
          result.scanItem.id,
        );

        successful.push(result);
      } catch (error) {
        failed.push({
          clientId: item.clientId,
          ean: item.ean,
          error: error instanceof Error ? error.message : 'Unknown error',
          errorCode: (error as any).code || 'UNKNOWN',
        });
      }
    }

    return {
      successful,
      duplicates,
      failed,
      totalProcessed: items.length,
      durationMs: Date.now() - startTime,
    };
  }

  async getStatus(batchId: string): Promise<BulkSyncStatus> {
    const batch = await this.batchesRepo.findById(batchId);
    if (!batch) throw new NotFoundException('SyncBatch', batchId);

    return {
      batchId: batch.id,
      status: batch.status as any,
      progress: {
        total: batch.totalItems,
        processed: batch.processedItems,
        succeeded: batch.succeededItems,
        failed: batch.failedItems,
        duplicates: batch.duplicateItems,
        percentage: batch.totalItems > 0 
          ? Math.round((batch.processedItems / batch.totalItems) * 100) 
          : 0,
      },
      startedAt: batch.startedAt || undefined,
      completedAt: batch.completedAt || undefined,
      errors: (batch.errors as BulkSyncError[]) || [],
    };
  }

  async cancel(batchId: string, userId: string): Promise<void> {
    const batch = await this.batchesRepo.findById(batchId);
    if (!batch) throw new NotFoundException('SyncBatch', batchId);
    if (batch.userId !== userId) {
      throw new BusinessException(ErrorCode.FORBIDDEN, 'Not your batch');
    }
    if (batch.status === 'completed' || batch.status === 'failed') {
      return; // Already done
    }

    // Remove from queue if still queued
    const jobs = await this.queue.getJobs(['delayed', 'waiting']);
    for (const job of jobs) {
      if (job.data.batchId === batchId) {
        await job.remove();
      }
    }

    await this.batchesRepo.update(batchId, {
      status: 'cancelled',
      completedAt: new Date(),
    });
  }
}
```

### 4. Bull Queue Processor

```typescript
// server/src/modules/scans/processors/bulk-scan.processor.ts
import { Processor, Process } from '@nestjs/bull';
import { Job } from 'bull';
import { Logger } from '@nestjs/common';
import { ScanSyncBatchesRepository } from '../repositories/scan-sync-batches.repository';
import { IdempotencyService } from '../services/idempotency.service';
import { ScanItemService } from '../services/scan-item.service';
import { BulkScanItemDto, BulkSyncError } from '../types/sync.types';

@Processor('bulk-scan')
export class BulkScanProcessor {
  private readonly logger = new Logger(BulkScanProcessor.name);

  constructor(
    private readonly batchesRepo: ScanSyncBatchesRepository,
    private readonly idempotency: IdempotencyService,
    private readonly itemService: ScanItemService,
  ) {}

  @Process('process-batch')
  async handleBatch(
    job: Job<{
      batchId: string;
      sessionId: string;
      userId: string;
      items: BulkScanItemDto[];
    }>,
  ): Promise<void> {
    const { batchId, sessionId, userId, items } = job.data;
    
    this.logger.log(`Processing batch ${batchId} with ${items.length} items`);

    await this.batchesRepo.update(batchId, {
      status: 'processing',
      startedAt: new Date(),
    });

    let processed = 0;
    let succeeded = 0;
    let failed = 0;
    let duplicates = 0;
    const errors: BulkSyncError[] = [];

    // Process in chunks of 100 for progress updates
    const chunkSize = 100;
    for (let i = 0; i < items.length; i += chunkSize) {
      const chunk = items.slice(i, i + chunkSize);
      
      for (const item of chunk) {
        try {
          const existing = await this.idempotency.findExisting(item.clientId, sessionId);
          if (existing) {
            duplicates++;
            processed++;
            continue;
          }

          const result = await this.itemService.recordScan(sessionId, item, userId);
          await this.idempotency.markProcessed(
            item.clientId,
            sessionId,
            result.scanItem.id,
          );
          succeeded++;
          processed++;
        } catch (error) {
          failed++;
          processed++;
          errors.push({
            clientId: item.clientId,
            ean: item.ean,
            error: error instanceof Error ? error.message : 'Unknown',
            errorCode: (error as any).code || 'UNKNOWN',
          });
        }
      }

      // Update progress every chunk
      await this.batchesRepo.update(batchId, {
        processedItems: processed,
        succeededItems: succeeded,
        failedItems: failed,
        duplicateItems: duplicates,
      });

      // Update job progress
      await job.progress(Math.round((processed / items.length) * 100));
    }

    // Finalize
    const finalStatus = failed > 0 ? 'partial' : 'completed';
    await this.batchesRepo.update(batchId, {
      status: finalStatus,
      completedAt: new Date(),
      errors: errors.slice(0, 100), // Cap at 100 to prevent huge JSON
    });

    this.logger.log(
      `Batch ${batchId} done: ${succeeded} ok, ${failed} failed, ${duplicates} dupes`,
    );
  }
}
```

## 🎯 Q&A Session

### Q1: Why client-generated UUIDs for idempotency?

**Expected Answer**:
- Mobile knows what it sent
- Same UUID on retry → server detects duplicate
- No round-trip needed for ID assignment
- Standard pattern (like Stripe idempotency keys)
- Works across network failures

---

### Q2: Why sync vs async threshold (50)?

**Expected Answer**:
- Small batches process in < 5 seconds
- HTTP timeout safe for sync
- Better UX (immediate response)
- Async overhead not justified
- > 50 needs async to avoid timeouts

---

### Q3: Why Redis for idempotency?

**Expected Answer**:
- Fast lookups (< 1ms)
- TTL built-in (7 days)
- Distributed (works across instances)
- Memory-efficient
- Falls back to DB for older items

---

### Q4: How to handle Redis failure?

**Expected Answer**:
- Idempotency service falls back to DB query
- DB has metadata.clientId index
- Slower but still correct
- Alert ops team
- Redis recovers, performance restored

---

### Q5: Why max 5000 items per batch?

**Expected Answer**:
- Reasonable upper bound
- Memory safety
- Single transaction limits
- Mobile devices unlikely to scan more offline
- Larger imports use BE-15 file upload

- Don't fail whole batch on one bad item

- Better UX than "all or nothing"
- User sees what worked
- Audit trail accurate

- Per-tenant queue limits (BE-32)

### Q8: How to scale to 100K scans/day?
