## 1. Architecture (the spine)

A single **curated manifest** is shared by mobile (bundled offline) and backend (DB seed). Honesty bar held
throughout: **no fabricated nutrition, no guessed EANs.**

- **Mobile bundles 29 product pack-shots** (WebP, ~0.81 MB total) → `apps/mobile/assets/v2/products/*.webp`,
  declared in `pubspec.yaml` as the dir `assets/v2/products/`. (4 fresh-produce items from the source set were
  excluded — no category/OFF fit. Source: `…/food farma app flow/IMG/IMG`. Promo banners in that folder are
  NOT yet wired — see §5.)
- **Browse** shows the bundled launch catalog instantly/offline, **merged** with server `/catalog` rows
  (real health rating, health-sorted, cursor-paginated) when reachable, de-duped by EAN.
- **Detail** is freemium: FREE = real health gauge (Mor reacts) + like/concern from real nutrition + key
  nutrients + Per-100g/50g sheet; PLUS-gated (blur+lock) = ingredient deep-dive + personalized For-You
  allergen flags. Honest "scan to unlock" ONLY when an EAN has no data yet.
- **Real data** flows in after the OFF seed resolves each product's real EAN → real nutrition/health.

## 3. CRITICAL FIX this session — CLI bootstrap

The 3 context-bootstrapping CLIs (import-curated / import-catalog / host-catalog-images) could **never run**:
their package.json scripts used **`tsx` (esbuild), which does NOT emit `emitDecoratorMetadata`** → Nest DI got
no constructor types → injected `undefined` → crash (`ConfigService … reading 'get'`).
**Fix:** scripts now use **`ts-node -r tsconfig-paths/register`** (tsc emits metadata; tsconfig-paths resolves `@/`).
`migrate.ts`/`reset.ts` stay on `tsx` (no Nest DI). **If you add another Nest-context CLI, use ts-node, not tsx.**

Needs Docker up (it is) + **stable outbound internet** to Open Food Facts.

```

cd ../apps/mobile
dart run tool/apply_resolved_eans.dart   # writes resolved_eans.g.dart with the real EANs
# rebuild the app
```

**Seed result in THIS sandbox:** bootstrapped + completed (exit 0), 8 categories ensured, but only **2/29 resolved**
(amul-ghee, britannia-white-bread — verified in Postgres) because OFF's **circuit breaker tripped** — the sandbox
throttles rapid outbound OFF calls so most short-circuited. **On a normal connection the seed resolves far more in
one run** (idempotent — safe to re-run). The pipeline itself is proven end-to-end.

## 7. Gotchas
- IDE edits files in parallel → **re-read before editing**; treat analyze/test/lint as source of truth.
- `resolved_eans.g.dart` is hand-applied by the dart tool (NOT build_runner) — don't expect codegen to touch it.
- `dart format` can split a braceless `if (...) return;` across lines → trips `curly_braces_in_flow_control_structures`;
  add braces if it does.
- New backend files often land with CRLF / condensed objects → run `pnpm exec eslint <path> --fix` (project is LF + Prettier).
- The OFF importer uses the product's REAL OFF name in the DB (e.g., Britannia bread shows as "sandwich white bread"
  in `products.name`); the app header shows the curated manifest name, so the UI stays clean.
